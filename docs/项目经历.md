# LightningDeal 秒杀系统 — 项目经历

## 技术栈

Spring Boot / MyBatis-Plus / MySQL / Redis / RabbitMQ / Elasticsearch / Vue 3 + ECharts / Docker

## 项目描述

高并发秒杀系统，解决**瞬时流量冲击下的限流、库存防超卖、异步削峰**核心问题。采用 Redis 预扣+Lua 原子操作实现入口控制，RabbitMQ 异步下单削峰，MySQL 乐观锁兜底库存，形成三层防护。集成 ES 搜索、WebSocket 实时推送、延迟队列自动取消订单、ECharts 实时大屏等配套能力，覆盖秒杀业务完整闭环。

## 个人贡献

1. **秒杀核心架构**：基于 Redis 令牌桶实现全局限流切面（`@RateLimit` 注解），500 并发压测限流率 97.4%、P99 延迟 126ms。Redis Lua 脚本原子校验库存+限购，MySQL 乐观锁兜底，确保零超卖。订单超时通过 RabbitMQ 延迟队列（`x-delayed-message`）30 分钟自动取消并回退全部资源。

2. **搜索与大屏**：搭建 ES + IK 分词搜索，DB 变更通过 MQ 异步同步到 ES（`TransactionSynchronizationManager` 事务提交后发送），3 次重试+死信队列不丢数据。基于 ECharts 实现实时监控大屏，展示 QPS 曲线、抢购流水、商品排行榜。

3. **全栈交付**：独立完成从 DB 设计到前端交互的全栈开发（活动管理/用户中心/订单系统/管理后台），自研 Python 多用户并发压测工具验证系统稳定性，Git 提交粒度清晰，开发文档覆盖每个功能的验收标准。
