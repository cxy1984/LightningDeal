## 二、已完成（本期）

### 订单超时自动取消 ✅

- **业务逻辑**：下单 30 分钟未支付 → 自动取消订单 → 恢复 Redis 库存 → 恢复 MySQL 库存 → 解除用户限购
- **技术方案**：RabbitMQ 延迟队列（`x-delayed-message` 插件）
- **实现链路**：
  ```
  下单成功
    ↓
  SeckillOrderConsumer.handleSeckillOrder()
    → 创建订单（status=0）
    → 发送延迟消息（30min）
    ↓ 30分钟后
  handleOrderTimeout(orderId)
    → orderService.cancelOrder(orderId)
    → Redis库存 +1
    → MySQL sold_stock -1
    → 用户购买计数 -1
    → 用户标记移除
    → 用户可重新抢购
  ```
- **修复**：
  - `cancelOrder()` 补充了 Redis 库存恢复和用户标记移除（之前只减了 MySQL sold_stock）
  - `refundOrder()` 同样补充了 Redis 库存恢复
- **延迟时间**：30 分钟（测试时改为 30 秒验证通过后改回）

## 四、RabbitMQ 延迟队列原理（当前任务）

```
下单成功
  │
  ├─→ 发送到 seckill.order.dlx.exchange（正常订单）
  │      TTL = 30 * 60 * 1000ms（30分钟）
  │      routingKey = order.timeout
  │      ↓
  │   seckill.order.dlx.queue（死信队列，无消费者）
  │      ↓ TTL 到期后路由到
  │   seckill.order.dead.exchange → seckill.order.dead.queue
  │      ↓
  │   SeckillOrderConsumer.handleDelayOrderTimeout()
  │      ↓
  │   检查订单状态，未支付则取消 + 恢复库存
```

当前 `RabbitMQConfig.java` 已配置好队列和绑定，需要实现 `handleDelayOrderTimeout()` 消费方法。
