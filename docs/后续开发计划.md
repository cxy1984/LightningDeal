### 1.2 Vite WebSocket 代理报错
- 现象：终端频繁输出 `[vite] ws proxy socket error: Error: read ECONNRESET`
- 原因：Vite 代理 `/ws` 到后端时，连接被重置（页面刷新、跳转时触发）
- **修复方案**（二选一）：

  **方案 A：前端直连后端 WebSocket（推荐）**
  ```js
  // vite.config.js - 去掉 ws 代理
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
    // 删掉 /ws 代理
  }
  ```
  ```js
  // stores/websocket.js - 直连后端
  // 开发环境直接连后端端口
  const wsUrl = process.env.NODE_ENV === 'development'
    ? `ws://localhost:8080/ws/seckill/result/${userId}`
    : `${protocol}//${window.location.host}/ws/seckill/result/${userId}`
  ```

  **方案 B：忽略错误（最低成本）**
  ```js
  // vite.config.js - 抑制 ws 代理错误日志
  server: {
    proxy: {
      '/ws': {
        target: 'ws://localhost:8080',
        ws: true,
        configure: (proxy) => {
          proxy.on('error', () => {});  // 忽略错误
        }
      }
    }
  }
  ```

## 三、详细任务说明

### P0-1：活动管理页面

**现状**：后端有 `POST /activity/create`、`PUT /activity/update` 接口，但前端没有对应的页面

**需要做**：
```
新页面: /admin/activity/create    → 创建活动表单
新页面: /admin/activity/edit/:id  → 编辑活动表单
修改页面: /activity/list          → 加上"管理"入口（管理员可见）
```

**表单字段**：
| 字段 | 类型 | 说明 |
|------|------|------|
| 活动名称 | 文本 | name |
| 商品名称 | 文本 | goodsName |
| 商品描述 | 文本域 | goodsDescription |
| 商品图片 | 图片上传 | goodsImage → 上传到 MinIO |
| 原价 | 数字 | originalPrice |
| 秒杀价 | 数字 | flashPrice |
| 总库存 | 数字 | totalStock |
| 限购数量 | 数字 | limitPerUser |
| 开始时间 | 日期选择 | startTime |
| 结束时间 | 日期选择 | endTime |

### P0-2：图片上传组件

**现状**：后端有 `POST /file/upload` API，前端 `api.uploadFile()` 已定义但未使用

**需要做**：

### P0-3：秒杀接口限流

**现状**：秒杀接口无防护，可以被刷

**实现方案**（Redis 令牌桶）：
```java
// 伪代码
public boolean tryAcquire(String key, int permits) {
    String script = "local key = KEYS[1] " +
                    "local rate = tonumber(ARGV[1]) " +
                    "local capacity = tonumber(ARGV[2]) " +
                    // ... 令牌桶 Lua 脚本
                    "return 1";
    Long result = redisTemplate.execute(script, ...);
    return result == 1;
}
```
