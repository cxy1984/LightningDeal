# 注册、登录、鉴权全链路总结

> 基于 Spring Security + JWT + BCrypt 的无状态认证方案

---

## 一、整体架构

```
┌────────────────────────────────────────────────────────────┐
│                        注册流程                            │
│                                                           │
│ 客户端 → POST /user/register → UserServiceImpl.register()  │
│   → BCrypt加密密码 → 存入MySQL → 返回用户信息               │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│                        登录流程                            │
│                                                           │
│ 客户端 → POST /user/login → UserServiceImpl.login()        │
│   → BCrypt校验密码 → 生成JWT Token → 返回前端               │
│                                                           │
│  前端收到Token后存在 localStorage                           │
└────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌────────────────────────────────────────────────────────────┐
│                     鉴权流程（每次请求）                    │
│                                                           │
│ 客户端 → 携带 Authorization: Bearer {token} → 请求         │
│   → JwtAuthenticationFilter                                │
│   → 解析Token → 验证签名 → 提取userId                      │
│   → 设置SecurityContext                                    │
│   → FilterSecurityInterceptor 检查路径权限                  │
│   → Spring MVC 控制器方法接收 Authentication 参数           │
└────────────────────────────────────────────────────────────┘
```

## 二、涉及文件清单

| 文件 | 作用 |
|------|------|
| `SecurityConfig.java` | Spring Security 核心配置：CORS、CSRF、Session 策略、路径权限、注册 JWT 过滤器 |
| `JwtUtil.java` | JWT Token 的生成、解析、验证工具类 |
| `JwtAuthenticationFilter.java` | 每次请求拦截，从 Header 提取 Token，解析后注入 SecurityContext |
| `UserController.java` | 注册/登录/获取用户信息 的 REST 接口 |
| `UserServiceImpl.java` | 注册（BCrypt 加密）、登录（BCrypt 校验 + 生成 Token）、查询用户 |
| `User.java` | 用户实体，含 username、password（加密）、phone、nickname |
| `LoginRequest.java` | 登录请求体（username + password） |
| `RegisterRequest.java` | 注册请求体（username + password + phone + email） |
| `DataInitializer.java` | 启动时初始化 admin、testuser 两个测试用户 |

## 三、密码加密（BCrypt）

### 3.1 配置

```java
// SecurityConfig.java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```

### 3.2 注册时加密

```java
// UserServiceImpl.register()
user.setPassword(passwordEncoder.encode(request.getPassword()));
// 存到数据库的是 BCrypt 密文，如：
// $2a$10$EixZaYVK1fsbw1ZfbX3OXePaWxn96p36PQm4sEPhMNPfF6dGqF8Za
```

### 3.3 登录时校验

```java
// UserServiceImpl.login()
if (!passwordEncoder.matches(request.getPassword(), user.getPassword())) {
    throw new BizException(401, "用户名或密码错误");
}
// 统一返回"用户名或密码错误"，不区分"用户不存在"和"密码错误"，防止枚举攻击
```

### 3.4 BCrypt 特点

| 特性 | 说明 |
|------|------|
| 自动加盐 | 每次加密结果不同，相同明文产生不同密文 |
| 防彩虹表 | 每个密码都带随机盐值 |
| 可调节强度 | 默认 10 轮哈希，可调高增加破解成本 |
| 行业标准 | Spring Security 默认推荐 |

## 四、JWT Token

### 4.1 配置

```yaml
# application.yml
jwt:
  secret: LightningDealSecretKey2024!@#$%^&*()LightningDealSecretKey
  expiration: 86400000  # 24小时（毫秒）
```

### 4.2 Token 生成（登录时）

```java
// JwtUtil.generateToken()
return Jwts.builder()
    .subject(userId.toString())           // sub = 用户ID
    .claim("username", username)           // 自定义字段：用户名
    .issuedAt(now)                         // 签发时间
    .expiration(new Date(now.getTime() + expiration))  // 过期时间
    .signWith(secretKey)                   // HMAC-SHA 签名
    .compact();
```

生成的 Token 示例：
```
eyJhbGciOiJIUzM4NCJ9.eyJzdWIiOiIxIiwidXNlcm5hbWUiOiJhZG1pbiIsImlhdCI6MTcxOT...
```

Token 三段结构：

| 段 | 示例 | 说明 |
|----|------|------|
| Header | `eyJhbGciOiJIUzM4NCJ9` | Base64 编码：`{"alg":"HS384"}` |
| Payload | `eyJzdWIiOiIxIiwidXNlcm5hbWUiOiJhZG1pbiJ9` | Base64 编码：`{"sub":"1","username":"admin"}` |
| Signature | 签名值 | 用 secretKey 对前两段签名，防篡改 |

### 4.3 Token 验证（每次请求）

```java
// JwtUtil.parseToken()
return Jwts.parser()
    .verifyWith(secretKey)         // 设置签名密钥
    .build()                       // 构建解析器
    .parseSignedClaims(token)      // 解析 + 验证签名
    .getPayload();                 // 获取 Payload（Claims）
```

验证失败（过期、篡改）直接抛 `JwtException`。

### 4.4 签名算法由 secret 长度决定

| secret 字节数 | 算法 |
|-------------|------|
| 32 ~ 47 | HMAC-SHA-256 |
| 48 ~ 63 | HMAC-SHA-384 |
| ≥ 64 | HMAC-SHA-512 |

当前 secret 58 字节 → 实际使用 **HMAC-SHA-384**。

## 五、鉴权过滤器（JwtAuthenticationFilter）

### 5.1 过滤器位置

```java
// SecurityConfig.java
.addFilterBefore(jwtAuthenticationFilter, UsernamePasswordAuthenticationFilter.class)
```

插入到 Spring Security 过滤器链的指定位置，在 `UsernamePasswordAuthenticationFilter` 之前执行。

### 5.2 执行流程

```
请求到达
  ↓
① extractToken(request)
  从 Header 取 "Authorization: Bearer {token}"
  bearerToken.startsWith("Bearer ") → true
  bearerToken.substring(7) → 得到纯 Token
  ↓
② jwtUtil.validateToken(token)
  parseToken() → 验证签名 + 检查过期
  成功 → 返回 true
  失败（过期/篡改）→ 返回 false
  ↓
③ 构建 UsernamePasswordAuthenticationToken
  principal = userId (Long)
  credentials = null
  authorities = Collections.emptyList()（无角色）
  setDetails(username)
  ↓
④ SecurityContextHolder.getContext().setAuthentication(authentication)
  注入 Security 上下文（ThreadLocal）
  ↓
⑤ filterChain.doFilter(request, response)
  放行，交给后续过滤器
```

### 5.3 提取 Token

```java
private String extractToken(HttpServletRequest request) {
    String bearerToken = request.getHeader("Authorization");
    if (StringUtils.hasText(bearerToken) && bearerToken.startsWith("Bearer ")) {
        return bearerToken.substring(7);  // 去掉 "Bearer " 前缀
    }
    return null;
}
```

## 六、SecurityConfig 配置详解

### 6.1 路径权限

```java
.authorizeRequests()
    .antMatchers("/swagger-ui.html", "/swagger-ui/**", "/v3/api-docs/**", "/v3/api-docs").permitAll()
    .antMatchers("/ws/**").permitAll()
    .antMatchers("/user/register", "/user/login").permitAll()
    .antMatchers("/activity/list", "/activity/detail/**", "/search/**").permitAll()
    .antMatchers("/dashboard/**").permitAll()
    .anyRequest().authenticated()     // ← 其他全部需要登录
```

从上到下匹配，命中即停：

| 请求路径 | 是否需要 Token |
|----------|--------------|
| `POST /user/login` | ❌ 不需要 |
| `GET /activity/list` | ❌ 不需要 |
| `GET /order/list` | ✅ **需要** |
| `DELETE /activity/1` | ✅ **需要** |
| `POST /seckill/execute` | ✅ **需要** |

### 6.2 异常处理

| 场景 | 返回 |
|------|------|
| 未登录或 Token 过期 | `401 + {"code":401,"msg":"未登录或 Token 已过期"}` |
| 权限不足（有 Token 但角色不够） | `403 + {"code":403,"msg":"权限不足"}` |

### 6.3 其他配置

| 配置 | 值 | 原因 |
|------|----|------|
| CORS | 允许所有来源 | 前端不同端口访问 |
| CSRF | 禁用 | 无状态 API，不使用 Session |
| Session | STATELESS | 完全无状态，不创建 Session |

## 七、控制器中获取用户

### 7.1 写法

```java
@GetMapping("/list")
public R<IPage<OrderVO>> list(Authentication authentication) {
    Long userId = (Long) authentication.getPrincipal();
    // userId = 当前登录用户的 ID
}
```

### 7.2 原理

Spring 在处理请求时，自动从 `SecurityContextHolder.getContext().getAuthentication()` 中取出 Authentication 对象，注入到控制器的参数中。

`authentication.getPrincipal()` 就是之前 `JwtAuthenticationFilter` 中设置的 `userId`。

## 九、当前存在的薄弱点

| 问题 | 说明 | 建议优先级 |
|------|------|-----------|
| 密码无强度校验 | 注册时可设任意短密码 | P1 |
| JWT secret 硬编码 | `application.yml` 明文存储，泄露可伪造 Token | P1 |
| Token 过期时间偏长 | 24h，泄露后窗口期长 | P2 |
| 无刷新 Token 机制 | 过期必须重新登录 | P2 |
| 无角色/权限体系 | 无法区分管理员和普通用户 | P2 |
| 无登出能力 | Token 无法主动失效 | P3 |
| `setDetails(username)` 不规范 | 标准应存 `WebAuthenticationDetails`，但不影响功能 | P3 |
